# Contributing to FMCG Data Engineering Project

Thank you for your interest in contributing to the FMCG Data Engineering Project! We welcome contributions to improve the data pipeline, add new features, or enhance documentation.

## How to Contribute

1.  **Fork the Repository**: Click the "Fork" button on the top right corner of the repository page.
2.  **Clone your Fork**:
    ```bash
    git clone https://github.com/your-username/project-de-fmcg-atlikon.git
    ```
3.  **Create a New Branch**:
    ```bash
    git checkout -b feature/your-feature-name
    ```
4.  **Make Changes**: Implement your changes or fixes.
5.  **Commit Changes**: Use descriptive commit messages.
    ```bash
    git commit -m "Add new feature: Description of change"
    ```
6.  **Push to GitHub**:
    ```bash
    git push origin feature/your-feature-name
    ```
7.  **Submit a Pull Request**: Go to the original repository and click "New Pull Request". Provide a clear description of your changes.

## Coding Standards

*   **Python/PySpark**: Follow PEP 8 guidelines. Use meaningful variable and function names.
*   **SQL**: Use uppercase for keywords (SELECT, FROM, WHERE) and indentation for readability.
*   **Notebooks**: Keep notebooks clean. Add markdown cells to explain the logic. Clear output before committing if the output is not necessary for understanding.

## Reporting Issues

If you find a bug or have a suggestion, please open an issue in the repository with details about the problem or feature request.
